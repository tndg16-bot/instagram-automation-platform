# 記事生成エンジン仕様書

## システム概要
320個の完全に一意な記事を生成する自動化システム

## 記事構成テンプレート

### 標準構造（1,500-3,000文字）
1. **導入（200-400文字）**
   - フックとなる問題提起
   - 記事の目的と対象読者
   - 解決する課題の提示

2. **本文（1,000-2,000文字）**
   - 3-5の主要セクション
   - 各セクション: 見出し + 内容 + 具体例
   - 箇条書き、テーブル、引用を活用

3. **結論（200-400文字）**
   - 主要ポイントの要約
   - アクショナブルな次のステップ
   - 関連記事への誘導

## 記事タイプ別テンプレート

### タイプ1: 解説・ガイド記事
```
- 導入: トピックの重要性
- 基礎知識: 概念の説明
- 詳細解説: ステップバイステップ
- 実践例: 具体的な活用シーン
- 注意点: よくある失敗と対策
- 結論: まとめと次のステップ
```

### タイプ2: 比較・対比記事
```
- 導入: 比較の必要性
- 比較対象の概要
- 詳細比較テーブル（5-8項目）
- ユースケース別推奨
- メリット・デメリット分析
- 選定の判断基準
- 結論: 結論と推奨
```

### タイプ3: 事例研究記事
```
- 導入: 事例の背景
- 課題の詳細
- 解決アプローチ
- 実施プロセス
- 結果と効果
- 学びと応用ポイント
- 結論: 教訓と実践のススメ
```

### タイプ4: トレンド分析記事
```
- 導入: 現状と変化
- 過去の推移
- 現在のトレンド
- データと統計
- 専門家の見解
- 将来予測
- 結論: 対応策と準備
```

### タイプ5: ハウツー・チュートリアル
```
- 導入: 達成できること
- 必要な準備
- Step 1: 詳細手順
- Step 2: 詳細手順
- Step 3: 詳細手順
- トラブルシューティング
- 上級テクニック
- 結論: 確認ポイントと次のステップ
```

### タイプ6: リスト・ランキング
```
- 導入: 選定基準
- 1位: 詳細解説
- 2位: 詳細解説
- 3位: 詳細解説
- 4-10位: 概要
- 比較サマリー
- 選び方ガイド
- 結論: 推奨とまとめ
```

### タイプ7: レビュー・評価
```
- 導入: レビュー対象の背景
- 概要と基本情報
- 詳細評価（5-8項目）
- 良いポイント
- 改善ポイント
- 価格・コスパ分析
- 対象ユーザー
- 結論: 最終評価と推奨
```

### タイプ8: 問題解決・対策
```
- 導入: 問題の影響
- 原因分析
- 解決策1: 即効性
- 解決策2: 中長期的
- 解決策3: 予防策
- 実践チェックリスト
- 注意点と副作用
- 結論: 実行計画
```

## SEO最適化ルール

### タイトル最適化
- キーワードを前に配置
- 数字を活用（「5つの」「完全ガイド」）
- 疑問形・命令形を活用
- 40文字以内を目安

### 本文最適化
- 見出しにキーワードを含める
- 最初の100文字に主要キーワード
- 適切なキーワード密度（1-2%）
- 内部リンク・外部リンクを適度に配置
- メタディスクリプションを設定

### 構造化データ
- H1: メインタイトル
- H2: 主要セクション（3-5個）
- H3: サブセクション
- 箇条書き、テーブル、引用を活用
- FAQセクション（該当する場合）

## 品質管理基準

### 必須チェック項目
- [ ] 文字数: 1,500-3,000文字
- [ ] 構成: 導入・本文・結論の明確な区分
- [ ] 具体性: 抽象的でない、具体例を含む
- [ ] 実用性: アクショナブルな情報
- [ ] 正確性: 正しいデータ・事実
- [ ] 一貫性: トピックからの脱線なし
- [ ] 可読性: 適切な段落長、見出し活用
- [ ] オリジナリティ: 他記事との重複チェック

### 品質レベル
- **レベルA**: 包括的、深い洞察、独自性あり
- **レベルB**: 充実した内容、実用的
- **レベルC**: 基本的な情報、標準的
